# Support

Thank you for using **CaeriusNet**! Here is where to look for help, in order of effectiveness.

## 1. Documentation

- [README](./README.md) — installation, quick start, public API surface.
- [Documentation site](https://caeriusnet.github.io/CaeriusNet/) (MkDocs) — long-form guides, examples, benchmarks.
- [CHANGELOG](./CHANGELOG.md) — what changed and when.
- [Source-generator diagnostics](./Documentations/diagnostics/) — `CAERIUS001…CAERIUS004` reference.

## 2. Search existing issues

Before opening a new issue, search:

- [Open issues](https://github.com/CaeriusNET/CaeriusNet/issues)
- [Closed issues](https://github.com/CaeriusNET/CaeriusNet/issues?q=is%3Aissue+is%3Aclosed)
- [Discussions](https://github.com/CaeriusNET/CaeriusNet/discussions) (if enabled)

Many usage questions have been answered already.

## 3. Open an issue

Use the appropriate template:

- [🐛 Bug report](https://github.com/CaeriusNET/CaeriusNet/issues/new?template=bug_report.yml) — something is broken or behaves unexpectedly.
- [💡 Feature request](https://github.com/CaeriusNET/CaeriusNet/issues/new?template=feature_request.yml) — propose a new capability.

Issues without a minimal reproducer will be labelled `needs-repro` and closed if no
follow-up is provided within 30 days.

## 4. Security vulnerabilities

**Do not** open a public issue. See [SECURITY.md](./SECURITY.md) for the
coordinated-disclosure process.

## 5. Commercial support

CaeriusNet is a community-maintained open-source project; no commercial support
contract is offered. Contributions and sponsorships via GitHub Sponsors are
welcome (see [`.github/FUNDING.yml`](.github/FUNDING.yml)).

## Response-time expectations

This is a volunteer project. We aim to triage non-security issues within
**7 calendar days** but make no formal SLA. Pull requests with tests and a clear
description merge fastest.
